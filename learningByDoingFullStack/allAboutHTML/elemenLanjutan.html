3. Attribute pada elemen input

Ini harus pakai att name dan browser chrome karena beda-beda setiap browser, sementara chrome = works

Attribute dalam pengiriman data(request-respond)

Ada dua jenis att dalam proses tersebut, yakni Att Action dan Method


Attribute Action

Action berfokus pada menentukan alamat tujuan pengiriman data. Terdapat URL/API yang dituju ke server yang domain sama atau beda

Tanpa att ini, data yang disubmit dan dikirim akan mengarah ke URL halaman tempat form berada

Ada dua jenis value/url yang ada pada att ini, Abolute URl dan Relatif URL

Absolute URL bertujuan untuk Mengarah ke alamat lengkap, termasuk protokol dan domain, misalnya "https://example.com/api/submit"

Intinya gunakan jenis ini untuk tujuan/server yang eksternal seperti API eksternal, layanan cloud, atau domain lain

Sementara Relative URL bertujuan untuk Mengarah ke path dalam domain yang sama, misalnya "/proses" atau "submit.php"

Jenis ini digunakan untuk server lokal atau halaman yang sama saat sedang testing misal

Attribute Method

Attribute method berfokus pada menentukan cara data dikirim ke alamat tujuan/server.

Ada dua jenis metode yang dapat digunakan pada form html, yakni GET dan POST

Sebelum memahami jenis keduanya, pahami dulu kerja HTTP.
HTTP merupakan protokol atau cara data dari request client dikirim oleh browser ke server dan cara resource hasil sebagai hasil jawaban yang diberikan oleh server melalui browser ke client
Dalam HTTP, terdapat dua bagian, yakni header yang berupa sebuah set berisi metadata(content-type, authorization, cookie, accept, content-lenght, dll) yang mana dibagi menjadi header request & header response.
Untuk lebih jelas bisa cek di: https://copilot.microsoft.com/shares/oSmWFenGPyYBZ8KyFDQe9
Selain itu, terdapat pula body yang merupakan konten/isi utama data yang dibagi menjadi body request dan body response dalam proses pengiriman data. Body Request adalah bagian dari HTTP request yang berisi data yang dikirim oleh client (biasanya browser) ke server.
Body Response adalah bagian dari HTTP response yang berisi data yang dikirim oleh server kembali ke client.
Keduanya berbentuk raw text seperti HTML, JSON, XML, Form-encoded data(application/x-www-form-urlencoded atau multipart/form-data)
Namun, body juga ada yang Empty Body atau tidak ada sama sekali isi yang biasanya ada di Mehtod GET karena browser tidak mengisi bagian body dari HTTP Request Client yang digant melalui query string dan ditambahkan ke URL. Contoh lebih lanjutnya sudah dibuktikan di sini: https://mdn.github.io/learning-area/html/forms/sending-form-data/get-method.html
Tapi, empty body juga ada pada proses response server karena server mengirim empty body pada saat tidak ada konten yang perlu dikembalikan seperti: response dengan status 204(No Content) atau response redirect 301, 302(biasanya hanya untuk memberi tahu browser pindah halaman).
Informsi lebih lanjut bisa dicek di: https://developer.mozilla.org/en-US/docs/Learn_web_development/Extensions/Forms/Sending_and_retrieving_form_data

Method GET digunakan untuk mendapatkan data dari server melalui browser yang terlebih dahulu dalam bentuk body request(empty body=diganti dalam bentuk query string dan ditambahkan di URL parameters).
URL parametersnya : pasangan key-value -> key=attribute name dan value=input user, tapi ada pelengkap(non-param) tanda tanya(?) sebagai pemisah path URL dan query string, ampersand(&) untuk memisahkan pasangan key-value. contoh implementasinya bisa cek di: https://mdn.github.io/learning-area/html/forms/sending-form-data/get-method.html Catatan: tidak semua data bisa ditambahkan ke URL menjadi string query karena bergantung pada protokol yang digunakan, jika http/https mendukung query string, tapi jika file:, data:, dll tidak bisa mendukung query string sehingga tidak cocok untuk method GET.

Method GET Implementation

Method Post digunakan untuk mengirim data dari browser ke server juga yang nanti akan menghasilkan resource dari response server.
Bedanya dengan method GET, dia punya body request yang non empty sehingga data tidak ditambahkan ke URL parameters.
Adanya body request bisa ditunjukan melalui URL yang tetap bersih serta sebagai POST method, dan pada header field(khususnya content-type & content-length) dalam header HTTP request yang bisa langsung di cek di sini: https://developer.mozilla.org/en-US/docs/Learn_web_development/Extensions/Forms/Sending_and_retrieving_form_data#:~:text=sebagai%20gantinya%3APOST-,HTTP,-Copy

C. Khusus Character

Se simple ada simbol karakter special/khusus yang biasa digunakan dalam web seperti copyright symbol(Â©) yang bukan merupakan standar kelompok ASCII characters.
ASCII characters hanya menyediakan karakter seperti huruf, nomor, dan beberapa simbol dasar lainnya.
Karakter lain juga ada yang ga bisa masuk HTMl walaupun ASCII sepreti (<) dan (>) karena dianggap sebagai tag
Solusinya dengan escaped character atau diganti dengan nilai numeric atau character reference pada karakternya.
Caranya ada dua, tetapkan nilai numeric(numeric entity) character/symbol tersebut atau dengan nama singkatan(named entity) yang telah ditetapkan oleh html 5. Penggunaan keduanya harus diawali dengan & dan diakhiri dengan ;
